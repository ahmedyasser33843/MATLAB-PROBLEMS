Part A Basic Operations 7 exercises
A.01
A.01easy
Order of operations + ans variable

Evaluate this expression and predict the result before running:

3 + 4 * 2 - (6/3)^2

What is stored in ans after running it? Now assign the result to x — does ans update? Why or why not?

MATLAB follows standard PEMDAS. ans only updates when a result is NOT assigned to a named variable. x = 5 does NOT update ans. Answer: 7.

Variables and arithmetic

Set x = 8.3 and y = 2.4. Compute:

  • (x² + y²) / (x − y)
  • √(x·y)
  • x^y
No implicit multiplication — 2x is a syntax error, write 2*x. Use sqrt() for square roots. The ^ operator is for powers.

P = 2 * pi * sqrt((a^2 + b^2) / 2)

Write this so a and b are defined silently (no output) but P prints. What is P?

Add a semicolon after a = 5; and b = 3; to suppress them. Leave P without a semicolon so it prints. Answer should be around 25.9.

Do both give the same results?

30° in radians = 30 * pi/180 = pi/6. Or just use sind(30) directly. All answers should be clean: 0.5, 0.5, 1.0.

All three should give the same answer. What is it?

The answer is 30°. All three methods are equivalent — just different ways to get there.

Euler's identity — floating point caveat

Euler's identity says e^(iπ)+1 = 0. Type exp(1i*pi) + 1. Is the result exactly zero?

Explain why not. Then write a proper numerical equality check using abs() and a tolerance.

The tiny leftover (~1.2e-16) is machine epsilon — floating point can't represent pi exactly. Never use == to compare floats. Use abs(result) < 1e-10 instead.

B.05hard
Reconstruct from polar form

A complex number has magnitude r=5 and angle θ=π/3. Reconstruct it in rectangular form using r * exp(1i * theta). Then verify by extracting abs() and angle() back — do you get r and θ?

Euler's formula: r·e^(iθ) = r·cos(θ) + i·r·sin(θ). Also works: r*cos(theta) + 1i*r*sin(theta). Both give the same thing.

Rows: [a; b; c]. Columns: [a' b' c'] — the apostrophe transposes a row vector into a column.

Inverse and determinant

For B = [1 3 3; 1 4 3; 1 3 4], compute det(B) and inv(B). Then verify by computing B * inv(B) — what should you get?

B * inv(B) should give the identity matrix eye(3). You'll likely see tiny floating-point errors like 1e-15 — use round() to clean it up.
C.05
C.05easy
Element-wise vs matrix operations

For A = [1 2 3; 4 5 6; 7 8 9] and B = [1 3 3; 1 4 3; 1 3 4]:

A ./ B    % element-wise division
A .* B    % element-wise multiply
A .^ 2    % square each element

Now try A ^ 2 (no dot). Different result?

The dot makes it element-wise. Without the dot, ^ is matrix power (A*A). Totally different results. This is one of the most common MATLAB bugs!
C.06
C.06medium
Verify matrix algebra properties

Use randi(10, 3, 3) to generate 3 random matrices A, B, C. Numerically verify:

  • A+B == B+A (commutative?)
  • A+(B+C) == (A+B)+C (associative?)
  • 5*(A+C) == 5*A + 5*C (distributive?)

Use isequal(). Then check: is A*B == B*A?

isequal() returns 1 (true) or 0 (false). Addition is commutative for matrices but multiplication generally is NOT — A*B ≠ B*A.

Upper triangular with identity diagonal (no loops)

Create A = randi(10,5,5). Produce a new matrix where:

  • All elements below the diagonal → zero
  • Diagonal → all ones
  • Above diagonal → kept from A

Do it in 1–2 lines. Use triu() and eye().

triu(A, 1) keeps only strictly above the diagonal (k=1 skips the diagonal itself). Then add eye(5) to put ones back on the diagonal.
C.08
C.08hard
Transpose properties — reversal rule

For random 3×3 matrices A and B, verify:

  • (AB)ᵀ = BᵀAᵀ  (note the reversed order!)
  • (A+B)ᵀ = Aᵀ + Bᵀ

Also confirm (AB)ᵀ ≠ AᵀBᵀ. Use norm(X − Y) < 1e-10 for equality checks.

Apostrophe ' transposes. norm() gives the "size" of the difference — near zero means equal. The order reversal in (AB)' = B'A' is the reversal rule — same applies to inverses.

Circuit analysis + conditioning

sin(0)/0 = 0/0 = NaN → gap in plot. Starting at 0.01 avoids this. The mathematical limit as t→0 of sin(t)/t is actually 1. Also use ./ for element-wise division.

In a 2×2 subplot, over x = linspace(0, 2*pi) plot:

  • cos(x)
  • sin(x)·cos(x)
  • sin²(x) + cos²(x)

What's special about the 4th subplot?

The 4th plot is a flat line at y=1 — that's the Pythagorean trig identity: sin²+cos²=1 for all x. Use subplot(2,2,n) where n is the panel number.

Sampling rate matters

Plot f(x) = 5·e²·sin(0.4x)·cos(4x) over [−20, 30] using:

  • plot with 100 points — correct?

  • plot with 1000 points — better?

Area between sin and cos over one period

Find the total area between sin(x) and cos(x) over [0, 2π]. You need to: find intersections, integrate |sin−cos| between each pair, sum. What's the total area?
